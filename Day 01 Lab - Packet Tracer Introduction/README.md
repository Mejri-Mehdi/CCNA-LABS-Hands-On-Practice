What I Learned
In this lab, I recreated a network topology shown in the Day 1 video (timestamp 16:40). I used Cisco Packet Tracer to build the following setup:

2 x Cisco 2911 routers

2 x Cisco 2960 switches

2 x Cisco 5505 firewalls

2 x PCs

2 x Servers

1 x Laptop (acting as an attacker)

I connected all devices using Packet Tracer’s “Automatically Choose Connection Type” feature – a quick way to let the tool select the correct cable (copper straight‑through, crossover, or serial).

Key takeaways
Basic network design – placing routers, switches, and firewalls correctly to segment traffic.

Device roles – routers for inter‑network routing, switches for local LAN connectivity, firewalls for security policies, and endpoints (PCs/servers) for services.

Attack simulation – adding an “attacker” laptop highlights the need for proper firewall rules and access controls.

Efficient cabling – using the automatic connection feature saves time and avoids manual cable type errors.

This hands‑on practice strengthened my understanding of how small enterprise networks are built in a simulator before moving to real hardware.
