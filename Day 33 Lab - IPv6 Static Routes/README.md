## What I Learned

In this lab, I worked with a network where **IPv6 addresses were pre‑configured on routers**, but PCs needed addressing via **SLAAC** (Stateless Address Autoconfiguration). I then configured **IPv6 static routes** with a backup path.

### Network information

| Link | IPv6 Network | Devices |
|------|--------------|---------|
| PC1 LAN (SW1) | 2001:db8:0:1::/64 | R1 G0/1, PC1 |
| R1 ↔ R3 | 2001:db8:0:13::/64 | R1 S0/0/0, R3 S0/0/1 |
| R1 ↔ R2 | (link-local only) | R1 S0/0/0, R2 S0/0/1 |
| R2 ↔ R3 | (link-local only) | R2 S0/0/0, R3 S0/0/1 |
| PC2 LAN (SW2) | 2001:db8:0:3::/64 | R3 G0/1, PC2 |

> **Note:** Serial connections between routers use **link‑local addresses only** – no global unicast addresses on those links.

### Tasks performed

1. **Enabled IPv6 routing on each router** – `ipv6 unicast-routing` on R1, R2, and R3.

2. **Configured PCs using SLAAC**:
   - On each PC, enabled IPv6 autoconfiguration (in Packet Tracer, set to **Auto** or use `ipv6 address autoconfig` in CLI).
   - **What IPv6 address was configured?** Each PC generated its IPv6 address using:
     - The network prefix from Router Advertisements (RAs) sent by the router (e.g., `2001:db8:0:1::/64`).
     - The interface ID generated via EUI‑64 (from the PC’s MAC address) or a random privacy address (depending on OS).
   - Verified with `ipconfig` (Windows) or `show ipv6 interface brief` (if using Cisco devices).

3. **Configured static routes for PC1 ↔ PC2 communication**:
   - **Primary path** (via R3, direct link):
     - On **R1**: `ipv6 route 2001:db8:0:3::/64 2001:db8:0:13::2` (next‑hop is R3’s serial global address)
     - On **R3**: `ipv6 route 2001:db8:0:1::/64 2001:db8:0:13::1`
   - **Backup path** (via R2) – with higher administrative distance:
     - On **R1**: `ipv6 route 2001:db8:0:3::/64 S0/0/0 FE80::<link-local-of-R2> 200` (AD 200, higher than default 1)
     - On **R3**: `ipv6 route 2001:db8:0:1::/64 S0/0/1 FE80::<link-local-of-R2> 200`
   - The **path via R2 should only be used if the direct R1‑R3 link fails**.

4. **Tested connectivity** – PC1 ping to PC2 succeeded via the primary path. After shutting down the R1‑R3 link, the backup route via R2 took over.

### Key takeaways

- **SLAAC** allows PCs to automatically obtain IPv6 addresses and default gateways from Router Advertisements (no DHCPv6 needed).
- The **prefix** comes from the RA; the **interface ID** is generated by the host (EUI‑64 or random).
- **Link‑local addresses** (FE80::/10) are used for neighbour discovery and can serve as next‑hop when global addresses aren’t configured.
- **Floating static routes** in IPv6 work the same way as IPv4 – a higher administrative distance (e.g., 200) makes them backup routes.
- When using a **link‑local next‑hop**, you must specify the **exit interface**.

show ipv6 interface brief
show ipv6 route
